# **项目目标**

本项目的名称为"**区块链非原子套利交易识别分析系统**"，项目旨在解决识别**去中心化交易所**（DEX）和**中心化交易所**（CEX）之间潜在的非原子套利行为，并计算潜在获利金额的问题。

具体而言，我们的目标是开发一个Web应用 ，该应用将实现以下核心目标：

+ **目标1： 异构数据源的获取与整合**
    - 本项目需要获取去中心化交易所 **Uniswap V3**  和中心化交易所 **Binance**  在特定时间范围内的特定交易对（USDT/ETH）的历史数据 ，这需要分别使用不同的 API 获取交易数据，并通过数据清洗、预处理与对齐进行整合。
+ **目标2： 实现价格趋势的可视化对比**
    - 本项目需要构建一个前端界面，以可视化的方式清晰地展示 **Uniswap V3** 和 **Binance** 在一定时间范围内的交易价格变化。
    - 用户应能够直观地对比两个市场在同一时间点的价格差异，帮助识别潜在的非原子套利机会。
+  **目标3：探索并实现非原子套利识别算法** 
    - 本项目将研究并实现一套方法，用于分析数据并识别潜在的非原子套利行为 。
    - 该方法可结合启发式规则、统计分析等多种手段，实现对非原子套利行为的精准识别。
+ **目标4： 量化潜在套利利润**
    - 对于识别出的每一组潜在非原子套利行为，我们的分析系统需要计算其潜在获利金额（以USDT为单位）。
    - 本项目应综合考虑交易量、滑点与交易手续费等多种因素， 以估算出更接近实际的净利润。  

本项目可量化的成果指标如下：

1. **数据完整性：**本项目应成功获取并处理特定时间段内 **Uniswap V3** 和 **Binance** 两个平台的 **95% **以上的有效成交数据。
2. **识别准确性：**本项目应在测试中成功识别 **80%** 以上的非原子套利行为。
3. **量化准确性：**本项目在测试中计算出的潜在套利利润应与实际净利润相差 **10%** 以内。

##### 版本控制：

1. 代码与文档仓库：

+ 在Github上创建私有仓库，将所有团队成员设为仓库协作者。
+ 通过Github的分支保护规则与成员权限设置，确保对main分支的合并需要代码审查和项目管理员批准，防止未授权的直接修改。

2. 工作流程：

+ 开发新功能与修复Bug时，成员从main分支创建一个新分支，在该分支上进行开发和提交。
+ 开发完成后，向main分支发起Pull Request，其他团队成员进行Code Review，通过审查后，由PM将PR合并到main分支。
+ 在main分支上，当完成一个里程碑或稳定版本时，PM创建Git Tag（如milestone-1-release）以标记版本。
+ 从第一个可交付成果初稿开始，所有对代码和文档的修改都通过git提交。
